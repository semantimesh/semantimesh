## 1. Executive Summary

**Проблема:** 3,7 млрд человек живут без стабильного интернета. В чрезвычайных ситуациях связь часто недоступна. IoT‑устройства тратят до 80 % энергии на передачу необработанных данных.

**Решение:** SemantiMesh — децентрализованная mesh‑сеть с гибридным сжатием данных и блокчейн‑стимулированием. Позволяет передавать сообщения и данные IoT без интернета с 15‑кратным сжатием.

**Ключевые технологии:**
* **BLE Mesh** для офлайн‑передачи данных (дальность 50–100 м);
* **Семантическое сжатие** (MobileBERT + zlib) для экономии энергии;
* **Блокчейн Polygon** для учёта активности нод и распределения токенов.

**Бизнес‑модель:**
* B2B‑лицензии для IoT‑платформ и корпораций;
* utility‑токен `$SEMANTI$` для оплаты услуг сети;
* стейкинг с доходностью 5–10 % APY.

**Ключевые метрики:**
* задержка передачи: $< 3$ с на 50 м;
* коэффициент сжатия: $\geq 15\times$ для IoT‑данных;
* число активных нод: $\geq 50\,000$ через 18 месяцев;
* MAU (месячная аудитория): $\geq 50\,000$;
* TVL (Total Value Locked) в стейкинге: $\geq \$5$ млн.

## 2. Проблема

**Статистика:**
* 3,7 млрд человек не имеют стабильного доступа к интернету (данные ITU, 2025);
* в ЧС до 90 % базовых станций сотовой связи выходят из строя;
* IoT‑устройства тратят 60–80 % энергии на передачу сырых данных;
* традиционные mesh‑сети имеют низкую пропускную способность;
* облачные IoT‑платформы зависят от инфраструктуры.

## 3. Решение

**SemantiMesh решает:**
* связь без интернета через BLE Mesh;
* экономию энергии за счёт семантического сжатия;
* масштабирование сети через блокчейн‑стимулирование.

**Принцип работы:**
1. Пользователь отправляет сообщение или данные IoT.
2. Данные сжимаются семантически (например, «температура 25 °C, влажность 60 %» → «temp:25, hum:60»).
3. Сжатые данные передаются через соседние ноды BLE Mesh.
4. Каждая нода‑ретранслятор получает микровыплату в токенах `$SEMANTI$`.
5. Получатель получает данные, система фиксирует транзакцию в блокчейне.

## 4. Технология

**Архитектура:**
* **Mesh‑слой:** BLE Mesh (Bluetooth 5.0+), дальность 50–100 м.
* **Семантический слой:** MobileBERT для текста, MobileNet для изображений.
* **Блокчейн‑слой:** Polygon (низкие комиссии, совместимость с Ethereum).

**Алгоритм сжатия:**
1. Семантическое обобщение: извлечение ключевых сущностей.
2. Традиционное сжатие: zlib к семантическим меткам.
3. Передача: упрощённый JSON с метками.

**Пример сжатия:**
* исходный текст: «Встреча состоится в 15:00 в кафе „У моста“ по адресу ул. Ленина, 15»;
* семантическая метка: «meet:15:00, loc:cafe_bridge»;
* сжатие: zlib → 10–15× уменьшение объёма.
